# 图书销售管理系统演示用例

本文档用于现场向助教演示系统功能。建议演示前先执行 `sql/schema.sql` 重置数据库，保证默认账号和演示数据一致。

## 1. 演示前准备

环境准备：

- 确认 MySQL 已启动。
- 确认 `src/main/resources/db.properties` 中的数据库账号密码正确。
- 在 MySQL 中执行 `sql/schema.sql` 初始化数据库。
- 在项目根目录执行 `mvn clean compile`，再执行 `mvn exec:java` 启动系统。

默认账号：

- 超级管理员：`admin / admin123`
- 普通管理员：`manager / 123456`

建议先说明：

- 系统采用 `Java Swing + MySQL + JDBC + Maven`。
- 密码不是明文保存，MD5 加密代码在 `src/main/java/com/bookstore/util/PasswordUtil.java`。
- 数据库脚本在 `sql/schema.sql`，可展示 `users`、`books`、`purchase_orders`、`sales_orders`、`finance_records` 等表。

## 2. 登录功能

用例目标：证明系统所有功能必须登录后才能使用，并且登录窗口不会记住密码。

操作步骤：

1. 启动系统，进入登录界面。
2. 输入错误密码，点击 `登录`。
3. 观察系统提示 `用户名或密码错误`。
4. 输入 `admin / admin123`，点击 `登录`。
5. 成功进入主界面。
6. 关闭系统后重新打开，观察用户名会保留上次登录账号，密码框为空。

预期结果：

- 错误密码无法登录。
- 正确账号可以登录。
- 系统只记住用户名，不记住密码。

## 3. 用户管理

用例目标：演示超级管理员和普通管理员的权限区别。

### 3.1 超级管理员创建普通管理员

操作步骤：

1. 使用 `admin / admin123` 登录。
2. 进入 `用户管理`。
3. 填写普通管理员信息，例如：
   - 用户名：`demo`
   - 新密码：`123456`
   - 确认新密码：`123456`
   - 真实姓名：`演示管理员`
   - 工号：`EMP-DEMO`
   - 性别：`男`
   - 年龄：`25`
4. 点击 `创建普通管理员`。
5. 在确认弹窗中点击 `是`。

预期结果：

- 系统提示普通管理员创建成功。
- 表格中出现新创建的普通管理员。
- 如果两次密码不一致，系统会提示两次输入的新密码不一致。

### 3.2 超级管理员重置或删除普通管理员

操作步骤：

1. 在 `用户管理` 表格中选中普通管理员。
2. 右键该用户。
3. 选择 `重置密码`，输入新密码和确认新密码。
4. 再次右键该用户，选择 `删除普通管理员`。
5. 在确认弹窗中点击 `否`，说明系统有防误操作确认。

预期结果：

- 超级管理员可以重置普通管理员密码。
- 超级管理员可以删除普通管理员。
- 不能对超级管理员执行删除或重置普通管理员密码操作。

### 3.3 普通管理员修改本人资料和密码

操作步骤：

1. 退出后使用普通管理员账号登录，例如 `manager / 123456`。
2. 进入 `用户管理`。
3. 修改真实姓名、工号、性别或年龄。
4. 如需修改密码，填写 `新密码` 和 `确认新密码`。
5. 点击 `修改本人信息`，在确认弹窗中点击 `是`。

预期结果：

- 普通管理员只能看到和维护自己的资料。
- 普通管理员可以修改自己的密码。
- 普通管理员不能创建、删除其他管理员，也不能查看所有用户资料。

## 4. 图书库存管理

用例目标：演示库存书籍维护、查询和图书信息修改。

### 4.1 查询图书

操作步骤：

1. 进入 `图书库存`。
2. 在关键词中输入 `Java`，点击 `查询`。
3. 再输入某个 ISBN、作者或出版社进行查询。

预期结果：

- 可以按编号、ISBN、书名、作者、出版社查询图书。
- 查询结果显示图书 ID、ISBN、书名、作者、出版社、零售价和库存。

### 4.2 新增图书

操作步骤：

1. 点击 `清空表单`。
2. 填写一本新书信息，例如：
   - ISBN：`978-demo-0001`
   - 书名：`演示新书`
   - 作者：`测试作者`
   - 出版社：`测试出版社`
   - 零售价：`66.00`
   - 库存：`10`
3. 点击 `新增图书`。
4. 在确认弹窗中点击 `是`。

预期结果：

- 系统提示新增成功。
- 表格中出现新书。

### 4.3 修改图书

操作步骤：

1. 在表格中选择一本图书。
2. 观察 ISBN 字段不可修改。
3. 修改书名、作者、出版社、零售价或库存。
4. 点击 `修改图书`。
5. 在确认弹窗中点击 `是`。

预期结果：

- ISBN 创建后不可修改。
- 其他图书信息可以修改。
- 修改前有确认弹窗，避免误操作。

## 5. 图书进货管理

用例目标：演示新书和已有图书的进货流程，以及未付款、已付款、已退货、已入库状态变化。

### 5.1 已有图书进货

操作步骤：

1. 进入 `进货管理`。
2. 在 `已有图书ID（新书可空）` 中输入已有图书 ID，例如 `1`。
3. 点击 `按ID载入图书`。
4. 填写：
   - 进货价：`60.00`
   - 数量：`5`
   - 入库零售价：`98.00`
5. 点击 `创建进货单`。

预期结果：

- 系统创建一条状态为 `unpaid` 的进货单。
- 已有图书信息会自动载入，减少手动输入错误。

### 5.2 新书进货

操作步骤：

1. 清空已有图书 ID。
2. 手动填写新书信息：
   - ISBN：`978-demo-0002`
   - 书名：`进货演示新书`
   - 作者：`进货作者`
   - 出版社：`进货出版社`
   - 进货价：`35.00`
   - 数量：`8`
   - 入库零售价：`58.00`
3. 点击 `创建进货单`。

预期结果：

- 系统创建一条状态为 `unpaid` 的进货单。
- 新书在未入库前不会进入图书库存。

### 5.3 付款、退货和入库

操作步骤：

1. 选择一条 `unpaid` 进货单，点击 `付款`。
2. 观察状态变为 `paid`，并生成财务支出记录。
3. 选择一条 `paid` 进货单，点击 `入库`。
4. 观察状态变为 `stocked`，库存增加。
5. 选择一条 `unpaid` 进货单，点击 `退货`。
6. 观察状态变为 `returned`。
7. 尝试对 `returned` 进货单点击 `入库`。

预期结果：

- 只有 `unpaid` 进货单可以付款或退货。
- 只有 `paid` 进货单可以入库。
- `returned` 进货单不能入库，系统会给出明确错误提示。

## 6. 图书销售

用例目标：演示按零售价销售图书，并自动扣减库存和生成收入记录。

操作步骤：

1. 进入 `图书销售`。
2. 输入关键词查询图书，例如 `Java`。
3. 选中一本库存大于 0 的图书。
4. 输入购买数量，例如 `1`。
5. 点击 `销售选中图书`。

预期结果：

- 系统提示销售成功。
- 图书库存数量减少。
- 财务账单中生成一条收入记录。
- 如果库存不足，系统会提示库存不足。

## 7. 财务账单

用例目标：演示系统能记录进货支出和销售收入，并按日期和类型查询。

操作步骤：

1. 进入 `财务账单`。
2. 设置开始日期和结束日期，格式为 `yyyy-MM-dd`。
3. 类型选择 `全部`，点击 `查询`。
4. 类型选择 `收入`，点击 `查询`。
5. 类型选择 `支出`，点击 `查询`。

预期结果：

- 进货付款会生成支出记录。
- 图书销售会生成收入记录。
- 可以按日期范围和账单类型筛选。

## 8. 数据库和加密说明

用例目标：回答助教关于数据库设计和密码加密的问题。

可展示内容：

1. 打开 `sql/schema.sql`，说明主要表：
   - `users`：用户和角色信息。
   - `books`：库存图书信息。
   - `purchase_orders`、`purchase_order_items`：进货单和进货明细。
   - `sales_orders`：销售记录。
   - `finance_records`：财务收入和支出记录。
2. 打开 `src/main/java/com/bookstore/util/PasswordUtil.java`。
3. 指出 `MessageDigest.getInstance("MD5")` 说明密码使用 MD5。
4. 打开 `src/main/java/com/bookstore/dao/UserDao.java`。
5. 指出登录和创建用户时调用 `PasswordUtil.md5(password)`，数据库保存的是 `password_hash`。

预期说明：

- 数据库结构覆盖了用户、库存、进货、销售、财务等实验要求。
- 用户密码不会明文保存。
- 财务记录由进货付款和图书销售自动产生。

## 9. 推荐现场演示顺序

建议按下面顺序演示，比较顺畅：

1. 登录失败和登录成功。
2. 超级管理员创建普通管理员。
3. 普通管理员登录并修改本人资料或密码。
4. 图书库存查询、新增、修改，说明 ISBN 不可修改。
5. 创建已有图书进货单并付款、入库。
6. 创建未付款进货单并退货。
7. 销售一本图书，观察库存减少。
8. 查看财务账单中的收入和支出。
9. 展示 `schema.sql` 和 `PasswordUtil.java`，说明数据库设计和 MD5 加密。
